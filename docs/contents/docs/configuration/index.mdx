Shellviz supports flexible configuration across all platforms (Python, JavaScript/Node.js, and Browser) through environment variables, constructor parameters, and framework-specific settings. This allows you to customize behavior across different environments and deployment scenarios.

## Configuration Overview

### Common Configuration Options

#### `SHELLVIZ_PORT`
- **Type**: Integer
- **Default**: `5544`
- **Description**: Port number for the Shellviz server

#### `SHELLVIZ_SHOW_URL` (Python/Node.js only)
- **Type**: Boolean
- **Default**: `true`
- **Description**: Whether to display the server URL on startup

#### `SHELLVIZ_URL`
- **Type**: String
- **Default**: `null` (constructs from port)
- **Description**: Custom base URL for the server (overrides port-based URL)

### Configuration Hierarchy

Configuration values are resolved in the following priority order (highest to lowest):

**Python:**
1. Constructor parameters
2. Django Settings (if available)
3. Environment Variables
4. Default Values

**JavaScript/Node.js:**
1. Constructor parameters  
2. Environment Variables (`process.env`)
3. Default Values

**Browser:**
1. Constructor parameters
2. Window Variables (`window.SHELLVIZ_*`)
3. Default Values

### Boolean Value Handling

For boolean configuration values:

**Python:** `true`, `1`, `yes`, `on` = true (all other values = false)  
**JavaScript:** `true`, `"true"`, `"1"`, `"yes"` = true (all other values = false)

## Quick Examples

### Basic Configuration
```bash
# Environment variables (Python/Node.js)
export SHELLVIZ_PORT=8080
export SHELLVIZ_URL="https://my-server.com"
```

```html
<!-- Window variables (Browser) -->
<script>
window.SHELLVIZ_PORT = 8080;
window.SHELLVIZ_URL = "https://my-server.com";
</script>
```

### Constructor Override
```python
# Python
from shellviz import Shellviz
sv = Shellviz(port=8080, url="https://my-server.com")
```

```javascript
// JavaScript/Node.js
import { ShellvizClient } from 'shellviz';
const sv = new ShellvizClient({port: 8080, url: "https://my-server.com"});
```

## Debugging Configuration

### Common Issues
- **Environment variables not loaded**: Verify variable names and restart your application
- **Boolean values not recognized**: Use `true`/`false` (not `True`/`False`)
- **URL connection failed**: Check if the server is running and accessible
- **Port conflicts**: Try a different port number

### Verification
Check your current configuration:
- **Python**: Access `sv.port`, `sv.base_url`, `sv.show_url_on_start` 
- **JavaScript**: Access `sv.port`, `sv.baseUrl`
- **Configuration values**: Import from `shellviz/config` or `shellviz.config`

For more detailed platform-specific configuration, see the individual library documentation pages linked above.
